Buatkan fitur Permanent Document Storage di dalam website dengan desain yang sama seperti dashboard utama: modern, premium, responsif, rapi, dark/light mode, warna biru–ungu–gelap, card rounded, sidebar folder, tabel dokumen, pencarian cepat, filter kategori, dan audit log. Fitur ini harus memungkinkan user mengunggah, menyimpan, mengelola, melihat, mengunduh, dan membagikan file dokumen secara aman dan permanen, seperti PDF, DOCX, XLSX, PPTX, TXT, ZIP, gambar, sertifikat, CV, portofolio, laporan, dan file penting lainnya. Buat halaman Document Vault dengan struktur folder, kategori, tag, metadata file, ukuran file, tipe file, tanggal upload, tanggal update, status akses, tombol preview, tombol download, tombol share, tombol rename, tombol move, tombol delete ke trash, dan restore file. Simpan file ke private storage bucket seperti Supabase Storage, bukan public bucket, lalu simpan metadata ke tabel database documents dengan field id, user_id, folder_id, file_name, original_name, file_type, file_size, storage_path, category, tags, description, visibility, version, checksum, is_favorite, is_archived, deleted_at, created_at, dan updated_at. Gunakan Row Level Security agar setiap user hanya bisa melihat dan mengelola dokumennya sendiri, sedangkan admin hanya bisa mengakses sesuai izin. Tambahkan fitur permanent archive agar dokumen tidak hilang setelah logout, refresh, atau deploy ulang, serta tambahkan versioning agar ketika user mengunggah file baru dengan nama sama, sistem tidak menimpa file lama tetapi membuat versi baru. Tambahkan fitur signed URL sementara untuk download, sehingga file tidak bisa diakses langsung tanpa izin, dan tambahkan opsi share link yang bisa kadaluarsa, memakai password, atau hanya bisa dibuka oleh email tertentu. Tambahkan preview aman untuk PDF dan gambar, serta fallback download untuk tipe file yang tidak bisa dipreview. Tambahkan fitur validasi file di frontend dan backend, batas ukuran file, daftar tipe file yang diizinkan, progress upload, retry upload, checksum/hash untuk memastikan file tidak rusak, toast notification, loading state, empty state, error state, dan konfirmasi sebelum delete permanen. Buat juga halaman Storage Dashboard yang menampilkan total file, total ukuran penyimpanan, file terbaru, file favorit, file terhapus, folder aktif, riwayat upload, dan aktivitas download. Semua aktivitas penting seperti upload, preview, download, share, rename, move, delete, restore, dan permanent delete harus dicatat ke tabel document_audit_logs. Untuk keamanan, gunakan private bucket, signed URL, RLS, role-based access control, input validation, malware scan placeholder, rate limiting, audit log, dan backup policy. Pastikan fitur ini terintegrasi dengan modul CV, HR access, direktori link, dan kalkulator, sehingga dokumen seperti CV, sertifikat, laporan kalkulator, dan file riset perusahaan bisa disimpan permanen, dicari, diunduh, dan dilindungi dengan akses yang jelas.
